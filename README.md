# Travel Wallet Bot

Telegram-бот для учета бюджета в поездках с конвертацией валют через `api.exchangerate.host` и хранением данных в `SQLite`.

## Возможности

- создание нескольких путешествий на пользователя;
- хранение пары валют (домашняя/поездки), курса и текущего баланса;
- автоматический пересчет расходов из валюты поездки в домашнюю валюту;
- подтверждение списания через inline-кнопки;
- история последних операций;
- ручная корректировка курса в активном путешествии.

## Архитектура

Проект построен по модульному принципу с разделением ответственности (Separation of Concerns):

- `bot.py` - слой представления и сценариев взаимодействия (Telegram handlers, FSM-подобные шаги ввода);
- `current_api.py` - инфраструктурный слой для интеграции с внешним API валют;
- `database.py` - слой доступа к данным (DAO-подход поверх `sqlite3`);
- `.env` - конфигурация секретов и токенов;
- `requirements.txt` - зависимости.

### Применяемые паттерны и принципы

- **Single Responsibility Principle (SRP):** каждый модуль решает свою отдельную задачу;
- **Layered Architecture:** UI (бот) -> сервисные вызовы API/БД -> хранение данных;
- **Repository/DAO style:** операции над таблицами инкапсулированы в `database.py`;
- **Fail Fast:** при отсутствии обязательных переменных окружения запуск прерывается сразу;
- **Defensive Programming:** валидация числового ввода и обработка ошибок API.

## Структура данных

### Таблица `trips`

Хранит состояния путешествий:
- пользователь (`user_id`);
- страны отправления/назначения;
- валютная пара (`home_currency`, `travel_currency`);
- курс (`rate`) и балансы в двух валютах;
- флаг активного путешествия (`is_active`).

### Таблица `expenses`

Хранит историю списаний:
- связь с путешествием (`trip_id`);
- сумма в валюте поездки и в домашней валюте;
- время создания записи.

## Требования

- Python `3.10+`
- доступ к интернету для запросов в `api.exchangerate.host`
- Telegram Bot Token от `@BotFather`
- API key для `api.exchangerate.host`

## Установка и запуск

1. Создайте виртуальное окружение:
   - Windows (PowerShell): `python -m venv venv`
2. Активируйте окружение:
   - `venv\Scripts\Activate.ps1`
3. Установите зависимости:
   - `pip install -r requirements.txt`
4. Создайте `.env` на базе `.env.example`:
   - `TELEGRAM_BOT_TOKEN=your_telegram_bot_token`
   - `CURRENCY_API_KEY=your_exchangerate_api_key`
5. Запустите бота:
   - `python bot.py`

## Команды и интерфейс

Slash-команды:
- `/start` - приветствие и главное меню;
- `/newtrip` - создание нового путешествия;
- `/switch` - переключение активного путешествия;
- `/balance` - просмотр текущего баланса;
- `/history` - история расходов;
- `/setrate` - ручное изменение курса.

Inline-меню дублирует ключевые действия и позволяет работать без slash-команд.

## Основной сценарий работы

1. Пользователь создает путешествие (страна отправления -> страна назначения).
2. Бот определяет валюты по словарю стран.
3. Бот получает курс через API и формирует стартовый баланс.
4. Любое сообщение-число считается расходом в валюте страны назначения.
5. Перед списанием бот показывает конвертацию и запрашивает подтверждение.
6. После подтверждения расход фиксируется в БД и уменьшает баланс.

## Конфигурация и безопасность

- секреты не хранятся в коде, только в `.env`;
- файл `.env` исключен из git через `.gitignore`;
- при отсутствии токенов приложение завершается с понятной ошибкой.

## Обработка ошибок

В проекте предусмотрены базовые проверки:
- неверный формат сумм/курсов (нечисловой ввод, отрицательные значения);
- отсутствие активного путешествия;
- ошибки внешнего API (включая неуспешный ответ сервиса).

Для production-режима рекомендуется добавить:
- retry/backoff для API-запросов;
- централизованный логгер;
- более точную типизацию и unit-тесты.

## Расширение проекта

Рекомендуемые направления развития:
- вынести бизнес-логику из `bot.py` в отдельный сервисный слой;
- заменить словарь стран на справочник ISO (`pycountry`);
- добавить миграции БД (например, `alembic`) при переходе на PostgreSQL;
- добавить тесты для `database.py` и `current_api.py`;
- внедрить CI-проверки (`ruff`, `pytest`, `mypy`).
